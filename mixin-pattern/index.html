Mixin Pattern

If you have a class dedicated only for methods that you want to use, but don't want to extend the class, that is where mixins come in.

In Vanilla JS however, this pattern applies to Objects.

Suppose you have an Object - A and some methods in its prototype.
You would like to use one or some of the methods from that Object - A, without making your current Object - B /Instance - B,
a new instance of Object - A. This is where the mixin pattern comes to the rescue.
